# Data Compression Bzip2 Project (Stage 1)

This project compiles and runs a simple Stage 1 pipeline from `stage1_simple.c`.

## Project Files

- `stage1_simple.c` - main C source file
- `Makefile` - build and run automation
- `input.txt` - input file used by default run target
- `output.txt` - output file generated by program
- `config.ini` - runtime configuration file

## Prerequisites

- GCC installed (`gcc` command available)
- Make tool available:
  - Linux/macOS: `make`
  - Windows: `mingw32-make` (or `make` alias mapped to it)

## Run the Project

From the project folder, run:

```powershell
make
```

Default `make` behavior in this project:

1. Compiles `stage1_simple.c`
2. Runs:
   - Windows: `.\stage1_simple.exe input.txt output.txt config.ini`
   - Linux/macOS: `./stage1_simple input.txt output.txt config.ini`

If successful, terminal output includes:

- `Stage 1 completed successfully.`
- `Output file created: output.txt`

## Windows: If `make` is Not Recognized

If PowerShell shows `make is not recognized`, use one of these options.

### Option A: Current terminal only

```powershell
Set-Alias -Name make -Value mingw32-make
make
```

### Option B: Permanent (all new PowerShell terminals)

```powershell
if (!(Test-Path $PROFILE)) { New-Item -ItemType File -Path $PROFILE -Force | Out-Null }
Add-Content $PROFILE "`nSet-Alias -Name make -Value mingw32-make"
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force
. $PROFILE
make
```
